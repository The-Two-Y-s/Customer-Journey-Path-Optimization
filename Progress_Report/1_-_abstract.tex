\begin{center}
    \large{\bf ABSTRACT}
\end{center}

This project addresses the problem of identifying optimal conversion paths in large-scale customer journey graphs. User interaction sequences are modelled as a directed weighted graph $G=(V,E)$, where edge weights encode the negative log-probability of each state transition, enabling maximum-probability path search to be solved as a shortest-path problem using Dijkstra's algorithm.

Two algorithms were implemented and evaluated. The \textbf{Baseline Dijkstra} algorithm provides globally optimal solutions in $O(|E|\log|V|)$ time. The \textbf{Probability-Pruned Dijkstra} extends the baseline with a threshold parameter $\tau$ that discards low-probability partial paths during traversal, reducing the effective search space to $O(|E'|\log|V|)$ with $|E'|\leq|E|$ while converging to the baseline as $\tau\to 0$.

Both algorithms were evaluated on synthetic graphs up to 10,000 nodes (2,160 experiment rows) across varying edge densities and probability distributions, and validated on two real-world e-commerce clickstream datasets: RetailRocket (2.7M events) and RecSys~2015 YOOCHOOSE (33M clicks), totalling 300 real-data experiment rows. Formal correctness proofs, theoretical complexity analysis, and comprehensive empirical evaluation were completed.

Results demonstrate that the pruned algorithm achieves median edge-count speedups ranging from $8.8\times$ ($\tau=0.001$) to $2{,}124\times$ ($\tau=0.5$), with wall-clock speedups reaching $881\times$. Crucially, the optimality gap is exactly $0.0000\%$ across all 171 path-found rows---the pruned algorithm either returns the exact same optimal path as the baseline or prunes it entirely, confirming the ``all-or-nothing'' property. On real-world data, wall-clock speedups reach $27{,}693\times$ (RetailRocket item-level) and $4{,}191\times$ (RecSys~2015).
