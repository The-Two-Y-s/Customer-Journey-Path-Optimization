\begin{center}
    \large{\bf ABSTRACT}
\end{center}

This project addresses the problem of identifying optimal conversion paths in large-scale customer journey graphs. User interaction sequences are modelled as a directed weighted graph $G=(V,E)$, where edge weights encode the negative log-probability of each state transition, reducing maximum-probability path search to a shortest-path problem solvable by Dijkstra's algorithm. Existing acceleration techniques---A* search, beam search, contraction hierarchies---either demand a problem-specific admissible heuristic, sacrifice optimality guarantees, or require heavy preprocessing ill-suited to behavioural data.

Two algorithms are proposed and evaluated. The \textbf{Baseline Dijkstra} algorithm provides globally optimal solutions in $O(|E|\log|V|)$ time. The \textbf{Probability-Pruned Dijkstra} extends the baseline with a threshold parameter $\tau$ that discards low-probability partial paths during traversal, reducing the effective search space to $O(|E'|\log|V|)$ with $|E'|\leq|E|$ while provably converging to the baseline as $\tau\to 0$.

Both algorithms were evaluated on synthetic graphs up to 10,000 nodes (2,160 experiment rows) and validated on two real-world e-commerce clickstream datasets: RetailRocket (2.7M events) and RecSys~2015 YOOCHOOSE (33M clicks), totalling 300 real-data rows. On synthetic benchmarks, wall-clock speedups range from $3.6\times$ ($\tau=0.001$) to $881\times$ ($\tau=0.5$), reducing median query time from 1.67\,ms to 0.002\,ms. On real-world item-level graphs, speedups reach $27{,}693\times$ (RetailRocket, reducing a 72\,ms query to 0.003\,ms) and $4{,}191\times$ (RecSys~2015). Crucially, the optimality gap is exactly $0.0000\%$ across all 250 path-found rows---the pruned algorithm either returns the globally optimal path or prunes it entirely, confirming a formal ``all-or-nothing'' property. These results suggest that threshold-based pruning offers a practical, parameter-controlled acceleration strategy for probabilistic journey optimization on graphs with tens of thousands of nodes, without sacrificing solution quality when paths are found.
